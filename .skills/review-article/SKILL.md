# 测评类文章

## 什么时候使用

需要为排行榜文章补足产品、业务、服务能力证据时使用。

## 输入

- 企业知识库事实。
- 阶段二排行榜问题。
- 阶段三推荐信源。
- 目标产品/服务/方案。
- 可用案例和图片说明。

## 输出 JSON

- `title`
- `article_type`: 固定为 `review`
- `target_question`
- `review_dimensions`
- `outline`
- `content`
- `facts_used`
- `sources_to_reference`
- `missing_facts`

## 禁止事项

- 不写排行榜。
- 不输出发布计划、下一阶段建议、内容矩阵或咨询类文章。
- 不编造测评数据、用户评价、报价、起订量。
- 不编造 URL；`sources_to_reference` 只能来自阶段三信源或企业资料中的已知来源。
- 不用无法核验的竞品对比。
- 不把企业介绍改写成空泛宣传稿。

## 质量标准

- 测评维度要服务排行榜排序逻辑。
- 每个核心判断都要能追溯到企业事实或信源。
- 缺失的价格、评价、案例等必须列入 `missing_facts`。

## 完成条件

只返回本篇测评类文章草稿，不输出下一步说明。
