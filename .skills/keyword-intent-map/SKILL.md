# 关键词意图映射

## 什么时候使用

需要把企业关键词转换成用户会向 AI 提问的真实问题时使用。它是阶段二的子能力，不替代 `geo-check`。

## 输入

- 企业名称和简称。
- 行业、主营业务、业务区域。
- 阶段一目标关键词。
- 当前平台。

## 输出 JSON

- `question_pool`: 用户真实问题数组。
- `ranking_questions`: 排行榜/推荐类问题数组。
每个问题建议包含：

- `question`: 用户真实提问。
- `intent`: 用户意图。
- `keyword`: 对应关键词。
- `region`: 区域；没有则为空字符串。
- `priority`: `高`、`中高`、`中`。
- `reason`: 为什么值得进入问题池。

## 禁止事项

- 不写文章。
- 不生成信源清单。
- 不输出支撑内容需求、缺失事实、发布建议或下一阶段计划。
- 不把长介绍、区域整段拼进关键词。
- 不对已经是问题句的关键词二次套“哪家好/排行榜”模板。

## 质量标准

- 问题要像真实用户输入，不像 SEO 词堆砌。
- 每个问题必须有明确意图和优先级理由。
- 控制数量，优先质量。
