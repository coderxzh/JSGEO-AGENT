# 排行榜问题池构建

## 什么时候使用

企业知识库已可用，用户确认进入阶段二时使用。豆包和 DeepSeek 必须分别生成，不合并平台结果。

## 输入

- 企业结构化 profile。
- 企业知识库摘要。
- 阶段一初始关键词。
- 当前平台：`doubao` 或 `deepseek`。

如需确认阶段边界，读取 `../references/geo-workflow.md`。

## 输出 JSON

只输出严格 JSON 对象，不要 Markdown 代码块：

- `summary`: 80 字以内，说明本轮问题池定位。
- `question_pool`: 用户真实问题数组，每项包含 `question`, `intent`, `keyword`, `region`, `priority`, `reason`。
- `ranking_questions`: 高优先级排行榜/推荐类问题数组，每项包含 `question`, `keyword`, `priority`, `why`。

## 禁止事项

- 不输出“为什么没进推荐”的泛诊断。
- 不输出支撑文章主题、内容矩阵、信源发现查询。
- 不输出缺失事实、下一步动作、发布建议。
- 不输出 3 个月执行节奏。
- 不输出标题、口播、详情页文案等成稿。
- 不输出泛化渠道方案。
- 不编造企业事实或第三方背书。

## 质量标准

- 排行榜问题必须是用户真实会问的高转化问题。
- 优先覆盖“哪家好、推荐、排行榜、供应商、品牌、公司、靠谱、怎么选”。
- 问题数量精简，默认 5 条真实问题和 5 条排行榜问题即可。
- 支撑内容需求必须服务于后续排行榜文章，不做无关营销内容。
